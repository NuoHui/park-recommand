## 🎓 架构设计亮点

### 1. 多层缓存设计

```
L1 缓存 (内存)          L2 缓存 (磁盘)
├─ 快速访问              ├─ 持久化
├─ 进程级                ├─ 跨进程
└─ Map 数据结构         └─ 文件系统

缓存键: "rec-{location}-{type}-{distance}"
过期策略:
  推荐结果: 24 小时
  LLM 响应: 1 小时
  POI 数据: 7 天
```

### 2. 并发控制和队列系统

```
RequestQueue:
├─ 优先级: HIGH > NORMAL > LOW
├─ 最大并发: 5
├─ 自动重试: 最多 2 次
├─ 请求去重: 防止重复
└─ 超时控制: 30s

并行执行:
  LLM 检查 ──┐
            ├─→ 并行处理
  地图查询 ──┘
```

### 3. 降级三层方案

```
完整推荐 (LLM + Amap)
    ↓ [失败]
【降级 #1】热门景点 (无 LLM 排序)
    ↓ [失败]
【降级 #2】缓存数据 (跨会话)
    ↓ [失败]
【降级 #3】模拟数据 (硬编码)
    ↓
最终返回某个结果 (保证用户体验)
```

### 4. 对话状态管理

```
对话阶段转移:
GREETING
  ↓
COLLECTING_LOCATION
  ↓
COLLECTING_TYPE
  ↓
COLLECTING_DISTANCE
  ↓
COLLECTING_DIFFICULTY
  ↓
QUERYING
  ↓
RECOMMENDING
  ↓
COMPLETED

每个阶段维持完整上下文和用户偏好累积
```
