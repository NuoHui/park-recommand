# 🧪 LLM 单元测试 - 快速开始

## 快速验证 LLM 是否正常工作

### ⚡ 最简单的方式（推荐）

```bash
npm run test:llm
```

这个命令会执行一系列快速测试，验证：
- ✅ 环境配置
- ✅ API 连接
- ✅ 基础调用
- ✅ 用户偏好提取
- ✅ 推荐决策生成
- ✅ 推荐解析

**预期输出（成功）：**
```

```

## 🔧 快速故障排除

### 1️⃣ 显示 "API Key not configured"

**问题**: 缺少 API Key 配置

**解决**:
```bash
# 创建 .env 文件
cat > .env << EOF
# LLM 提供商选择：openai 或 anthropic
LLM_PROVIDER=openai

# OpenAI 配置（如果使用 OpenAI）
OPENAI_API_KEY=sk-your-key-here
OPENAI_MODEL=gpt-4o-mini

# 或者 Anthropic 配置（如果使用 Anthropic）
# ANTHROPIC_API_KEY=sk-ant-your-key-here
# ANTHROPIC_MODEL=claude-3-5-sonnet-20241022
EOF
```

### 2️⃣ 显示 "Could not authenticate"

**问题**: API Key 无效或已过期

**解决**:
1. 检查 API Key 是否完整（通常以 `sk-` 开头）
2. 登录 OpenAI 或 Anthropic 平台检查配额
3. 重新生成 API Key 并更新 `.env` 文件

### 3️⃣ 显示 "Connection timeout"

**问题**: 网络连接问题

**解决**:
```bash
# 方法 1: 检查网络
ping api.openai.com

# 方法 2: 增加超时时间（在 .env 中添加）
LLM_TIMEOUT=120000

# 方法 3: 检查是否需要代理
export HTTP_PROXY=http://proxy:8080
export HTTPS_PROXY=https://proxy:8080
npm run test:llm
```

### 4️⃣ 显示 "Model not found"

**问题**: 使用的模型名称不正确或不可用

**解决**:
- OpenAI: 使用 `gpt-4o-mini`、`gpt-4-turbo` 等
- Anthropic: 使用 `claude-3-5-sonnet-20241022`、`claude-3-opus-20240229` 等
