# 集成测试：宝安西乡公园推荐场景

## 📋 测试概述

本集成测试验证系统能否正确处理以下用户需求：

> **用户需求**: "推荐深圳宝安西乡附近的公园，距离不限制。"

该测试覆盖完整的推荐系统工作流程，从用户输入解析到最终推荐结果输出。

## 🎯 测试目标

1. **完整流程验证**: 验证用户输入 → LLM 分析 → 地图查询 → 结果排序 → 推荐输出的完整链路
2. **功能完整性**: 确保系统正确识别位置、类型、距离等关键信息
3. **数据准确性**: 验证推荐结果包含必要的地理信息和原因说明
4. **性能基线**: 建立推荐系统的性能基线

## 🧪 测试用例设计

### Test 1: 地点解析 - 宝安西乡坐标获取

**功能**: 地址编码（Geocoding）

**操作**:
- 输入: 地址"宝安西乡"，城市"深圳"
- 调用: `AmapClient.geocode()`
- 验证: 返回有效的经纬度坐标

**预期输出**:
```json
{
  "name": "宝安西乡",
  "city": "深圳",
  "longitude": 113.8863,
  "latitude": 22.6054,
  "formattedAddress": "广东省深圳市宝安区西乡..."
}
```

**验证条件**:
- ✅ 获取正确的坐标信息
- ✅ 地址格式化正确
- ⏭️ 跳过: 当 AMAP_API_KEY 未配置

---

### Test 2: 类型识别 - 识别"公园"关键词

**功能**: 用户输入意图识别

**操作**:
- 输入: 用户查询 "推荐深圳宝安西乡附近的公园，距离不限制。"
- 调用: `LLMEngine.extractUserPreference()`
- 验证: 正确识别景点类型为"公园"(park)

**预期输出**:
```json
{
  "type": "park",
  "confidence": 0.95
}
```

**验证条件**:
- ✅ 提取的景点类型正确
- ✅ 识别置信度 ≥ 0.5
- ⏭️ 跳过: 当 LLM API Key 未配置

---

### Test 3: 距离偏好识别 - 无限制距离

**功能**: 距离偏好提取

**操作**:
- 输入: 用户查询中的距离信息 "距离不限制"
- 调用: `LLMEngine.extractUserPreference()`
- 验证: 正确识别为无限制距离

**预期输出**:
```json
{
  "distancePreference": "unlimited",
  "maxDistance": null
}
```

**验证条件**:
- ✅ 距离偏好识别为"无限制"
- ✅ maxDistance 为 null 或很大的值
- ⏭️ 跳过: 当 LLM API Key 未配置

---

### Test 4: 地图查询 - 获取宝安西乡周边公园

**功能**: POI 搜索

**操作**:
- 输入: 关键词"公园"，区域"深圳"
- 调用: `AmapClient.searchPOI()`
- 验证: 返回有效的公园列表

**预期输出**:
```json
{
  "parks": [
    {
      "name": "莲花山公园",
      "location": {
        "longitude": 114.0681,
        "latitude": 22.5461
      },
      "address": "深圳市福田区...",
      "type": "公园"
    },
    // ... 更多公园
  ],
  "parkCount": 5
}
```

**验证条件**:
- ✅ 返回至少 1 个公园信息
- ✅ 每个公园包含位置和地址信息
- ⏭️ 跳过: 当 AMAP_API_KEY 未配置

---

### Test 5: 完整推荐流程 - 宝安西乡公园推荐

**功能**: 端到端推荐系统

**操作**:
1. 初始化 `DialogueManager`
2. 模拟用户输入:
   - 地点: "宝安西乡"
   - 类型: "p" (公园)
   - 距离: "4" (无限制)
3. 调用: `DialogueManager.getRecommendations()`
4. 验证: 返回推荐列表

**预期输出**:
```json
{
  "success": true,
  "recommendations": [
    {
      "id": "uuid",
      "name": "莲花山公园",
      "reason": "这是深圳最知名的公园之一，游玩时间约 2-3 小时",
      "distance": 5.2,
      "rating": 4.8
    },
    // ... 更多推荐
  ],
  "recommendationCount": 5,
  "averageRecommendationScore": 0.85
}
```

**验证条件**:
- ✅ 推荐成功 (success === true)
- ✅ 返回至少 1 条推荐
- ✅ 每条推荐包含: id, name, reason
- ✅ 推荐有序排列
- ⏭️ 跳过: 当 LLM 或地图 API Key 未配置

---

### Test 6: 结果验证 - 推荐内容完整性检查

**功能**: 推荐结果质量验证

**操作**:
- 输入: 前置测试生成的推荐结果
- 验证: 检查字段完整性和有效率
- 要求: 至少 80% 的推荐是有效的

**验证标准**:
```
必需字段:
- name: 公园名称
- location: 位置信息
- reason: 推荐原因
- type: 景点类型

有效率计算:
有效推荐数 / 总推荐数 ≥ 80%
```

**预期输出**:
```json
{
  "totalRecommendations": 5,
  "validRecommendations": 5,
  "invalidRecommendations": 0,
  "validationRate": "100%"
}
```

## 📊 测试执行

### 前置条件

1. 环境配置完成
   - AMAP_API_KEY 已设置
   - LLM_PROVIDER 已设置 (openai 或 anthropic)
   - 对应 API Key 已配置

2. 依赖服务可用
   - 高德地图 API 可访问
   - LLM 服务 (OpenAI/Claude) 可访问
   - 网络连接正常

### 运行测试

```bash
# 运行集成测试
npm run test:integration

# 或作为完整测试的一部分
npm run test

# 运行特定的集成测试过滤器
npm run test -- --filter integration
```
