# LLM 单元测试指南

本文档介绍如何使用单元测试验证 LLM 系统是否正常工作。

## 📋 快速开始

### 运行快速测试

最简单的方法是使用快速测试脚本：

```bash
npm run test:llm
```

这将运行一系列测试来验证：
- ✅ 环境配置
- ✅ LLM 连接
- ✅ 基础调用
- ✅ 用户偏好提取
- ✅ 推荐决策生成
- ✅ 推荐解析

### 运行完整单元测试

使用完整的测试运行器：

```bash
npm run test:unit
```

或运行所有测试：

```bash
npm run test
```

## 🧪 测试内容

### 1. 环境配置验证

检查必要的环境变量是否正确配置。

**检查项：**
- LLM 提供商设置 (OpenAI 或 Anthropic)
- API Key 是否存在
- 模型名称是否指定

**示例错误：**
```
❌ 缺少 OPENAI_API_KEY 配置
```

**解决方案：**
1. 检查 `.env` 文件
2. 确保 `OPENAI_API_KEY` 或 `ANTHROPIC_API_KEY` 已设置
3. 确保 API Key 有效且未过期

### 2. LLM 连接测试

验证是否能连接到 LLM 服务。

**测试内容：**
- 创建 LLM 客户端
- 发送测试消息到 LLM API
- 验证收到正常响应

**示例错误：**
```
❌ 连接失败: Could not authenticate with API Key provided
```

**解决方案：**
1. 检查 API Key 是否有效
2. 检查网络连接（可能有代理或防火墙阻止）
3. 检查 API 配额是否已用尽
4. 尝试更换 VPN 或网络环境

### 3. 基础调用测试

测试能否成功调用 LLM 进行简单任务。

**测试内容：**
- 调用 LLM 生成一句话介绍深圳
- 验证响应内容和 Token 使用

**示例输出：**
```
ℹ️  响应内容（前 100 字）:
深圳是中国改革开放的先行地和试验田，是国家经济特区，位于广东省南部...
ℹ️  Token 使用: 45
✅ 基础调用成功！
```

### 4. 用户偏好提取测试

测试 LLM 能否从用户输入中提取关键信息。

**测试内容：**
- 输入：「我想在宝安找一个适合登山的景点，距离不超过 10 公里」
- 验证提取的信息（位置、景点类型、距离偏好等）

**预期输出：**
```
意图: provide_info
置信度: 0.95
提取信息: {
  "location": "宝安",
  "parkType": "hiking",
  "maxDistance": 10
}
✅ 用户偏好提取成功！
```

### 5. 推荐决策生成测试

测试 LLM 能否基于用户偏好生成搜索参数。

**测试内容：**
- 输入：用户偏好（位置、类型、距离等）
- 输出：搜索参数和推荐理由

**预期输出：**
```
应该推荐: true
搜索参数: {
  "location": "宝安",
  "parkType": "hiking",
  "maxDistance": 10,
  "keywords": ["登山", "爬山"]
}
置信度: 0.85
✅ 推荐决策生成成功！
```

### 6. 推荐解析测试

测试 LLM 能否解析推荐结果并提取结构化信息。

**测试内容：**
- 输入：推荐景点列表（JSON 格式）
- 输出：解析后的景点信息和推荐理由

**预期输出：**
```
解析出 2 个景点
说明: 根据您的偏好，我为您推荐以下登山景点...
✅ 推荐解析成功！
```

## 🔧 环境配置

### 基础配置

创建 `.env` 文件，配置 LLM 提供商：

#### OpenAI 配置

```env
# LLM 提供商
LLM_PROVIDER=openai

# OpenAI 配置
OPENAI_API_KEY=sk-your-api-key-here
OPENAI_MODEL=gpt-4o-mini
```

#### Anthropic Claude 配置

```env
# LLM 提供商
LLM_PROVIDER=anthropic

# Anthropic 配置
ANTHROPIC_API_KEY=sk-ant-your-api-key-here
ANTHROPIC_MODEL=claude-3-5-sonnet-20241022
```

### 可选配置

```env
# LLM 参数
LLM_TEMPERATURE=0.7          # 0-2, 越高越随机
LLM_MAX_TOKENS=2000          # 最大回复 token 数
LLM_TOP_P=1.0                # 0-1, 核采样参数
LLM_TIMEOUT=60000            # 超时时间（毫秒）

# 代理设置（如果需要）
HTTP_PROXY=http://proxy.example.com:8080
HTTPS_PROXY=https://proxy.example.com:8080
```

## 🐛 常见问题

### Q: API Key 验证失败

**A:** 检查以下项：
1. API Key 是否复制完整（通常以 `sk-` 开头）
2. API Key 是否有效和未过期
3. 检查 `.env` 文件中的大小写和空格
4. 如果使用了代理，确保代理配置正确

### Q: 连接超时

**A:** 
1. 检查网络连接
2. 尝试增加超时时间：在 `.env` 中设置 `LLM_TIMEOUT=120000`
3. 检查是否有防火墙阻止
4. 尝试更换网络或 VPN

### Q: 模型不存在

**A:**
1. 检查模型名称是否拼写正确
2. 确认该模型在你的账户中可用
3. 对于 OpenAI，常见模型：
   - `gpt-4o-mini` (推荐)
   - `gpt-4-turbo`
   - `gpt-3.5-turbo`
4. 对于 Anthropic，常见模型：
   - `claude-3-5-sonnet-20241022` (推荐)
   - `claude-3-opus-20240229`
   - `claude-3-sonnet-20240229`

### Q: 响应质量差

**A:**
1. 调整温度参数（`LLM_TEMPERATURE`）
2. 增加 `max_tokens` 以获得更长的响应
3. 检查 system prompt 是否合适
4. 尝试不同的模型

### Q: 特定测试失败，但连接成功

**A:**
1. 检查模型是否支持该功能
2. 查看日志获取详细错误信息
3. 可能需要调整 prompt 或参数
4. 尝试手动调用来排查问题

## 🚀 高级用法

### 运行特定的单元测试

```bash
# 只运行单元测试
npm run test:unit

# 只运行集成测试
npm run test:integration

# 运行性能测试
npm run test:perf
```

### 查看详细日志

LLM 操作会输出详细日志，位置：`logs/` 目录

```bash
# 查看最新日志
tail -f logs/app.log
```

### 扩展测试

要添加新的测试用例，编辑 `src/__tests__/unit/llm.test.ts` 文件：

```typescript
tests.push(
  (async () => {
    try {
      // 你的测试代码
      logger.info('✅ 测试 X 通过', { /* 数据 */ });
      return {
        name: '测试名称',
        passed: true,
      };
    } catch (error) {
      logger.error('❌ 测试 X 失败', { error });
      return {
        name: '测试名称',
        passed: false,
        error: error instanceof Error ? error.message : String(error),
      };
    }
  })()
);
```
