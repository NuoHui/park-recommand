# 系统架构概览

## 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    CLI 交互层 (CLI Layer)                    │
│  - 用户输入处理  - 结果展示  - 交互流程管理                  │
└────────────────┬────────────────────────────────────────────┘
                 │
┌────────────────┴────────────────────────────────────────────┐
│              对话管理层 (Dialogue Layer)                      │
│  - 多轮对话维护  - 状态机管理  - 上下文保存                  │
└────────────────┬────────────────────────────────────────────┘
                 │
┌────────┬───────┴──────┬───────────────────┬────────────────┐
│        │               │                   │                │
│   LLM 引擎        地图服务              缓存系统        推荐服务
│  (LLM Engine)   (Map Service)      (Cache System)   (Recommender)
│        │               │                   │                │
└────────┴───────┬──────┴───────────────────┴────────────────┘
                 │
┌────────────────┴────────────────────────────────────────────┐
│                  底层服务层 (Service Layer)                  │
│  - API 调用  - 数据处理  - 错误处理  - 日志记录              │
└─────────────────────────────────────────────────────────────┘
```

## 核心模块

### 1. CLI 层 (`src/cli/`)

**职责**:
- 解析命令行参数
- 管理用户交互流程
- 格式化输出结果
- 处理用户输入

**主要文件**:
- `index.ts` - CLI 框架入口
- `commands/` - 命令定义
- `prompts/` - 交互提示

**关键特性**:
- 基于 Commander.js 的命令框架
- 结构化的交互流程
- 美观的输出格式化

### 2. 对话管理层 (`src/dialogue/`)

**职责**:
- 维护多轮对话上下文
- 管理对话状态
- 提取用户意图
- 生成后续问题

**主要文件**:
- `manager.ts` - 对话管理器
- `state-machine.ts` - 状态机定义
- `context.ts` - 对话上下文

**关键特性**:
- 状态机驱动的对话流
- 完整的上下文保存
- 用户意图识别

### 3. LLM 引擎 (`src/llm/`)

**职责**:
- 接口调用 LLM API（OpenAI/Claude）
- Prompt 工程和优化
- 结果解析和验证
- 错误恢复

**主要文件**:
- `client.ts` - LLM 客户端封装
- `engine.ts` - LLM 引擎逻辑
- `prompts.ts` - Prompt 模板
- `parser.ts` - 结果解析

**支持的 LLM**:
- OpenAI (GPT-4, GPT-3.5)
- Anthropic (Claude)

### 4. 地图服务 (`src/map/`)

**职责**:
- 调用高德地图 API
- 地点搜索和地理编码
- 距离计算
- 数据清洗和转换

**主要文件**:
- `service.ts` - 地图服务逻辑
- `amap.ts` - 高德 API 集成

**支持的功能**:
- 地理位置编码
- 周边搜索
- 距离计算（批量）
- 路线规划

### 5. 缓存系统 (`src/cache/`)

**职责**:
- 两层缓存管理（内存 + 磁盘）
- 缓存过期管理
- 缓存去重
- 缓存统计

**主要文件**:
- `manager.ts` - 缓存管理器
- `storage.ts` - 本地存储

**缓存策略**:
- **内存缓存**: 高频数据（LRU 淘汰）
- **磁盘缓存**: 持久化数据（JSON 存储）
- **TTL 管理**: 自动过期清理

### 6. 推荐服务 (`src/services/`)

**职责**:
- 整合各模块功能
- 生成推荐列表
- 推荐排序和过滤
- 结果组织

**主要流程**:
1. 接收用户偏好
2. 调用地图服务获取候选地点
3. 调用 LLM 进行智能排序
4. 返回推荐结果

## 数据流

### 推荐流程时序图

```
用户输入
   │
   ▼
CLI 层 (解析输入)
   │
   ▼
对话管理 (维护状态)
   │
   ├─→ LLM 引擎 (决策)
   │
   ├─→ 地图服务 (获取数据)
   │     │
   │     └─→ 缓存系统 (查询缓存)
   │
   ├─→ 推荐服务 (综合排序)
   │
   └─→ CLI 层 (格式化输出)
       │
       ▼
      输出结果
```

### 缓存流程

```
请求
 │
 ▼
内存缓存 ─→ 命中 ─→ 返回
 │
 X (未命中)
 │
 ▼
磁盘缓存 ─→ 命中 ─→ 返回
 │
 X (未命中)
 │
 ▼
API 调用 ─→ 缓存结果 ─→ 返回
```

## 关键设计模式

### 1. 模块化设计

- 清晰的模块边界
- 各模块独立开发和测试
- 通过接口解耦

### 2. 分层架构

- **表示层**: CLI 交互
- **业务层**: 对话、推荐、决策
- **服务层**: API 调用、数据处理
- **数据层**: 缓存、存储

### 3. 插件式架构

- LLM 提供商可插拔（OpenAI/Claude）
- 地图服务可扩展
- 缓存策略可定制

### 4. 事件驱动

- 异步处理各个模块
- 错误恢复机制
- 日志追踪

## 性能优化点

### 1. 缓存优化

- **两层缓存**: 内存 + 磁盘，减少 API 调用
- **智能预热**: 启动时加载热门数据
- **过期管理**: 自动清理过期缓存

### 2. 并发优化

- **Promise.all**: 批量处理请求
- **连接复用**: HTTP 连接复用
- **队列管理**: 控制并发数

### 3. 响应优化

- **流式响应**: 逐个返回结果
- **增量加载**: 先返回部分结果
- **预缓存**: 启动时预加载数据

## 扩展点

### 2. 新增地图服务

### 3. 自定义推荐算法

在 `src/services/recommender.ts` 中修改推荐逻辑。

### 4. 新增缓存策略

在 `src/cache/manager.ts` 中实现新的缓存策略。
