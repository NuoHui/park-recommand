✅ 实现的功能
═══════════════════════════════════════════════════════════════════════════════

  ✓ LLM 信息充分性检查 (shouldRecommend)
    → 验证用户偏好信息是否完整
    → 返回缺失信息或通过信号
  
  ✓ LLM 搜索参数优化 (generateSearchParams)
    → 理解用户隐含意图
    → 生成最优搜索关键词
    → 提供搜索决策和置信度
  
  ✓ 地图 API 景点查询 (searchRecommendedLocations)
    → 调用高德 POI 搜索 API
    → 自动计算景点距离
    → 按距离过滤结果
  
  ✓ LLM 推荐排序和理由生成 (parseRecommendations)
    → 计算景点相关性评分
    → 生成自然语言推荐理由
    → 排序景点优先级
  
  ✓ 推荐格式化输出 (formatRecommendations)
    → 转换为标准推荐格式
    → 按相关性排序
    → 限制最多 5 个推荐
  
  ✓ 三层降级处理
    → 第一层: 热门景点列表
    → 第二层: 模拟数据
    → 第三层: 错误提示
  
  ✓ 完整的错误处理
    → Try-catch 包裹所有 API 调用
    → 异常自动恢复
    → 友好的错误提示


🔄 推荐流程 (7 步)
═══════════════════════════════════════════════════════════════════════════════

  1️⃣ 前置检查
     ✓ 确保处于 QUERYING 阶段
     ✓ 初始化 LLMService 和 LocationService
  
  2️⃣ LLM 信息检查
     ✓ 验证位置、景点类型、距离是否齐全
     ✓ 不足时返回缺失信息
  
  3️⃣ LLM 参数优化
     ✓ 理解用户需求
     ✓ 生成最优搜索参数和关键词
  
  4️⃣ 地图 API 查询
     ✓ 搜索符合条件的景点
     ✓ 计算用户到景点的距离
  
  5️⃣ LLM 排序解析
     ✓ 为景点生成相关性评分
     ✓ 生成自然语言推荐理由
  
  6️⃣ 格式化输出
     ✓ 转换为最终推荐格式
     ✓ 按相关性排序，最多 5 个
  
  7️⃣ 返回结果
     ✓ 状态转移到 RECOMMENDING
     ✓ 返回推荐列表
